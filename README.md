## Repository structure
Each package folder is:

- self-contained
- internally consistent
- structured the same way where possible

This makes it easy to:

- compare tools
- cross-reference concepts
- reuse patterns

---

## Structure of each package folder

Each package folder generally contains:

- **README.md**  
  High-level explanation of the tool, its purpose, and how it fits into real projects

- **docs/**  
  One or more long-form handouts explaining concepts, rules, pitfalls, and mental models

- **src/** / **tests/** / **examples/**  
  Realistic, minimal code used to practice and reinforce ideas

Not every folder will have every subdirectory, but the intent remains consistent.

---

## How to use this repository

### First time learning a package
1. Read the package's README.md
2. Read the handout in docs/
3. Run and inspect the example code
4. Modify or extend examples if needed

### Reviewing a package later
1. Skim the handout top-to-bottom
2. Jump to specific sections as needed
3. Open example files to refresh muscle memory

### Reusing patterns in real projects
1. Copy structures, not code blindly
2. Adapt patterns to context
3. Use this repo as a design reference
